## 1. 개요 (Overview)

본 프로젝트는 웹 브라우저에서 수집한 유용한 링크를 효율적으로 저장하고 관리할 수 있는 북마크 관리 웹 애플리케이션입니다. HTML, CSS, JavaScript만을 사용한 클라이언트 사이드 애플리케이션으로, LocalStorage를 활용하여 별도의 백엔드 서버 없이 데이터를 영구적으로 저장합니다.

### 프로젝트 배경
현대의 웹 사용자들은 학습, 업무, 개발 과정에서 수많은 유용한 링크를 접하게 됩니다. 하지만 브라우저의 기본 북마크 기능은 체계적인 관리가 어렵고, 나중에 필요한 링크를 찾는 데 많은 시간이 소요됩니다.

## 2. 문제 정의 (Problem Statement)

### 핵심 문제
- **정보 손실**: 유용한 링크를 발견해도 저장하지 않거나, 저장해도 나중에 찾지 못하는 경우가 빈번함
- **비효율적인 검색**: 크롬이나 다른 브라우저의 북마크는 폴더 구조만으로 정리되어 검색 및 필터링이 불편함
- **시간 낭비**: 이전에 봤던 튜토리얼, 문서, 레퍼런스를 다시 찾기 위해 구글 검색을 반복하는 비효율 발생
- **메모 부족**: 링크만 저장되고 당시 왜 저장했는지, 어떤 내용인지 기록할 방법이 없음

### 사용자 페인 포인트
1. "분명히 봤던 블로그 글인데 어디에 저장했는지 모르겠어요"
2. "북마크가 100개가 넘어가니까 정리가 안 돼요"
3. "링크만 저장되어 있어서 나중에 보면 뭔지 기억이 안 나요"
4. "카테고리별로 나눠서 보고 싶은데 크롬 북마크는 너무 불편해요"

### 측정 가능한 문제
- 평균적으로 저장한 링크를 다시 찾는데 **1-5분** 소요
- 찾지 못해 포기하고 구글에 다시 검색하는 경우 발생

## 3. 목표 (Goals & Objectives)

### 프로젝트 목표
**저장한 링크를 10초 이내에 재발견할 수 있는 직관적인 북마크 관리 시스템 구축**

### 세부 목표
1. **효율성**: 링크 저장 및 검색 시간을 기존 대비 80% 단축
2. **사용성**: 별도의 학습 없이 5분 내에 모든 기능 사용 가능
3. **접근성**: 어떤 기기에서든 접근 가능한 반응형 웹 앱
4. **지속성**: LocalStorage를 활용한 영구 데이터 저장
5. **확장성**: 향후 추가 기능 구현이 용이한 모듈형 코드 구조

### 학습 목표 (교육 과정)
- HTML/CSS/JavaScript 기본기 향상
- DOM 조작 및 이벤트 처리 실습
- LocalStorage API 활용
- 사용자 중심 UI/UX 디자인 경험
- Git/GitHub를 통한 버전 관리 및 배포

## 5. 핵심 기능 (Features - MoSCoW 우선순위)

| 기능 | 설명 | 우선순위 |
|------|------|----------|

| **북마크 목록 표시** | 저장된 북마크를 카드 형식의 그리드 레이아웃으로 표시<br>- 제목 클릭 시 새 탭에서 링크 열기<br>- URL, 카테고리, 메모, 생성일 표시 | P0 |
| **북마크 삭제** | 각 북마크의 삭제 버튼으로 영구 삭제<br>- 삭제 전 확인 다이얼로그 표시<br>- LocalStorage에서 데이터 제거 | P0 |
| **실시간 검색** | 검색창에 키워드 입력 시 즉시 필터링<br>- 검색 대상: 제목, URL, 메모<br>- 대소문자 구분 없음 | P0 |
| **카테고리 필터** | 카테고리 드롭다운으로 필터링<br>- 기본 카테고리: 개발, 디자인, 뉴스, 유틸리티, 기타<br>- "전체" 옵션으로 모든 북마크 표시 | P0 |
| **북마크 수정** | 저장된 북마크의 정보 수정 가능<br>- 모든 필드 수정 가능<br>- 수정 시 updatedAt 필드 갱신 | P0 |
| **정렬 기능** | 4가지 정렬 옵션 제공<br>- 최신순 (생성일 내림차순)<br>- 오래된순 (생성일 오름차순)<br>- 제목순 (가나다순)<br>- 카테고리순 | P0 |
| **데이터 영구 저장** | LocalStorage를 사용한 클라이언트 사이드 저장<br>- 브라우저 종료 후에도 데이터 유지<br>- JSON 형식으로 직렬화 | P0 |

**사용자 경험 향상을 위해 구현이 권장되는 기능**

| 기능 | 설명 | 우선순위 |
|------|------|----------|

| **반응형 디자인** | 다양한 화면 크기에 최적화<br>- 모바일 (< 480px)<br>- 태블릿 (480px - 768px)<br>- 데스크톱 (> 768px) | P1 |

**리소스가 허락한다면 구현할 수 있는 기능**

## 7. 성공 지표 (Success Metrics)

### 정량적 지표

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| **링크 재탐색 시간** | 10초 이내 | 사용자 테스트로 검색부터 링크 클릭까지 소요 시간 측정 |
| **기능 구현 완성도** | 100% (모든 Must Have 구현) | 체크리스트 기반 확인 |
| **선택 과제 구현** | 최소 1개 이상 | 즐겨찾기 또는 Export/Import 구현 |
| **페이지 로딩 속도** | 3초 이내 | Chrome DevTools Performance 탭 |
| **코드 품질** | 주석 포함, 함수 모듈화 | 코드 리뷰 |

### 정성적 지표
- **사용성**: 처음 사용하는 사람도 5분 내에 모든 기능 이해 및 사용 가능
- **직관성**: 별도의 설명 없이도 UI만으로 기능 파악 가능
- **안정성**: 예외 상황(빈 입력, 잘못된 URL 등)에서도 오류 없이 작동
- **디자인**: 시각적으로 깔끔하고 현대적인 UI

### 테스트 시나리오
1. **기본 CRUD 테스트**
   - 북마크 10개 추가 → 모두 목록에 표시되는지 확인
   - 특정 북마크 수정 → 변경 사항이 반영되는지 확인
   - 특정 북마크 삭제 → 목록에서 제거되는지 확인

2. **검색 및 필터 테스트**
   - "React" 검색 → 제목/URL/메모에 "React" 포함된 항목만 표시
   - "개발" 카테고리 필터 → 개발 카테고리만 표시
   - 검색 + 필터 동시 적용 → 두 조건 모두 만족하는 항목만 표시

3. **정렬 테스트**
   - 최신순 정렬 → 최근 추가한 북마크가 먼저 표시
   - 제목순 정렬 → 가나다순으로 정렬

4. **데이터 영속성 테스트**
   - 북마크 5개 추가 → 브라우저 종료 → 재접속 → 데이터 유지 확인

5. **반응형 테스트**
   - 모바일 화면 (375px) → 레이아웃 정상 표시
   - 데스크톱 화면 (1920px) → 그리드 레이아웃 정상 작동

## 9. 위험 요소 및 대응 방안 (Risks & Mitigations)

### 기술적 위험

| 위험 요소 | 영향도 | 가능성 | 대응 방안 |
|-----------|--------|--------|-----------|
| **LocalStorage 용량 초과** | 중 | 낮음 | - 북마크 개수 제한 (예: 최대 500개)<br>- 용량 경고 메시지 표시<br>- Export 기능으로 백업 유도 |
| **브라우저 호환성 문제** | 중 | 중 | - ES6+ 기능 사용 시 호환성 확인<br>- Can I Use 사이트 참고<br>- Polyfill 적용 (필요시) |
| **데이터 손실 (LocalStorage 삭제)** | 높음 | 중 | - Export 기능으로 백업 권장<br>- 앱 시작 시 백업 안내 메시지<br>- 삭제 시 재확인 다이얼로그 |
| **XSS 공격 (악의적인 스크립트 입력)** | 중 | 낮음 | - 입력값 이스케이프 처리<br>- innerHTML 대신 textContent 사용 |

## 10. 데이터 구조 (Data Schema)

### Bookmark 객체 구조

```javascript
{
  id: "1638123456789",              // String: timestamp 기반 고유 ID
  title: "React 공식 문서",          // String: 북마크 제목 (필수)
  url: "https://react.dev",         // String: 웹사이트 URL (필수)
  category: "개발",                  // String: 카테고리 (필수, 고정 옵션)
  description: "React 학습용",       // String: 메모/설명 (선택)
  createdAt: "2025-11-15T10:30:00", // ISO String: 생성 날짜
  updatedAt: "2025-11-15T10:30:00", // ISO String: 수정 날짜
  isFavorite: false                  // Boolean: 즐겨찾기 여부 (선택 과제)
}
```

```javascript
// Key: "bookmarks"

{
  "bookmarks": "[{...}, {...}, ...]"
}
```
